## Sprint 1: Initial deployment

**Objective:** Establish the basic server infrastructure and get the application running on the VPS.

<Steps>
  <Step title="Initial Droplet setup">
    Access the server via the **DigitalOcean Console** and update the system:

    ```bash
    # Update system packages
    sudo apt update && sudo apt upgrade -y

    # Install Node.js 18.x
    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt install -y nodejs

    # Install PM2 globally
    sudo npm install -g pm2

    # Install Git
    sudo apt install -y git
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/devcarlosperez/CanAccesible.git
    cd CanAccesible
    ```
  </Step>

  <Step title="Frontend setup">
    ```bash
    cd frontend
    npm install
    ```
  </Step>

  <Step title="Backend setup">
    Install dependencies and create the production environment file:

    ```bash
    cd ../backend
    npm install
    nano .env.production
    ```

    Add these environment variables to `.env.production`:

    ```env
    # Node Environment
    NODE_ENV=production

    # Server Port
    PORT=85

    # Database Configuration
    DB_HOST=your-db-cluster.db.ondigitalocean.com
    DB_PORT=25060
    DB_USER=doadmin
    DB_PASSWORD=your_db_password
    DB_NAME=canaccesible

    # DigitalOcean Spaces Configuration
    DO_SPACE_NAME=your-space-name
    DO_REGION=fra1
    DO_ACCESS_KEY=your_spaces_access_key
    DO_SECRET_KEY=your_spaces_secret_key

    # Frontend URL
    FRONTEND_URL=http://<DROPLET_IP>:5173
    ```
  </Step>

  <Step title="Database initialization">
    ```bash
    # Run migrations
    NODE_ENV=production npx sequelize-cli db:migrate

    # Run seeders
    NODE_ENV=production npx sequelize-cli db:seed:all
    ```
  </Step>

  <Step title="Start applications with PM2">
    **Frontend:**
    ```bash
    pm2 start "NODE_ENV=production npm run dev -- --host 0.0.0.0" --name frontend
    ```

    **Backend:**
    ```bash
    pm2 start "NODE_ENV=production node index.js" --name backend
    ```

    **Save configuration and enable startup on reboot:**
    ```bash
    pm2 save
    sudo pm2 startup
    ```
  </Step>

  <Step title="Verify deployment">
    Check that both processes are running:

    ```bash
    pm2 status
    ```
  </Step>
</Steps>

## Sprint 2: Production hardening

**Objective:** Transition from a development setup to a production-ready configuration, securing environment variables and connecting to the managed database.

<Steps>
  <Step title="Environment variables">
    Edit `backend/.env.production` and add the following secrets:

    ```env
    # Email Configuration (Resend)
    RESEND_API=re_your_resend_api_key

    # Security Secrets
    JWT_SECRET=Canaccesible_ultra_super_secret_key_2025
    SESSION_SECRET=Canaccesible_ultra_super_secret_session_key_2025

    # DigitalOcean Spaces
    DO_ACCESS_KEY=your_access_key
    DO_SECRET_KEY=your_secret_key
    DO_SPACE_NAME=images-cruds
    DO_SPACE_ENDPOINT=fra1.digitaloceanspaces.com
    ```

    Update `frontend/.env.production` to point `VITE_API_URL` to the production IP.
  </Step>

  <Step title="CORS configuration">
    Update `backend/index.js` to restrict requests to known origins only:

    ```javascript
    // backend/index.js
    const allowedOrigins = [
      "http://localhost:5173",
      "http://localhost:85",
      "http://<DROPLET_IP>:5173",
    ];
    ```
  </Step>

  <Step title="Database reset and seeding">
    Reset the database for a clean production state:

    ```bash
    # 1. Drop existing tables
    npx cross-env NODE_ENV=production sequelize-cli db:drop

    # 2. Run migrations
    npx cross-env NODE_ENV=production sequelize-cli db:migrate

    # 3. Seed initial data
    npx cross-env NODE_ENV=production sequelize-cli db:seed:all
    ```
  </Step>
</Steps>

## Sprint 3: Domain, proxy & security

**Objective:** Make the application accessible via `canaccesible.es`, secure it with SSL/TLS, and implement Nginx as a reverse proxy.

<Steps>
  <Step title="Domain and DNS">
    - Acquire `canaccesible.es` via Nominalia.
    - Configure **A Records** in the DNS panel to point `@` and `www` to the Droplet's public IP (`64.227.124.155`).
  </Step>

  <Step title="Install Nginx">
    ```bash
    sudo apt install nginx
    ```
  </Step>

  <Step title="Configure Nginx reverse proxy">
    Create `/etc/nginx/sites-available/canaccesible.es`:

    ```nginx
    server {
        listen 80;
        server_name canaccesible.es www.canaccesible.es;

        # Proxy to Frontend (Vite Dev Server)
        location / {
            proxy_pass http://localhost:5173;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        # Proxy to Backend API
        location /api/ {
            proxy_pass http://localhost:85;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }
    }
    ```

    Enable the site:

    ```bash
    sudo ln -s /etc/nginx/sites-available/canaccesible.es /etc/nginx/sites-enabled/
    sudo rm /etc/nginx/sites-enabled/default
    sudo systemctl restart nginx
    ```
  </Step>

  <Step title="Install SSL/TLS certificate">
    ```bash
    sudo apt install certbot python3-certbot-nginx
    sudo certbot --nginx -d canaccesible.es -d www.canaccesible.es
    ```

    Certbot automatically configures Nginx to listen on port 443 (HTTPS) and redirects all HTTP traffic to HTTPS.
  </Step>

  <Step title="Update environment variables for domain">
    Edit `backend/.env.production` to reflect the new domain:

    ```env
    FRONTEND_URL=https://canaccesible.es
    ```

    Also update the CORS configuration in `backend/index.js` to allow the new domain.
  </Step>

  <Step title="Database maintenance">
    Perform another round of database reset (drop → migrate → seed) to ensure data integrity with the new domain configuration.
  </Step>
</Steps>

## Sprint 4: Optimization & Dockerization

**Objective:** Optimize frontend performance by serving static files, containerize the LDAP service, and finalize real-time communication features.

<Steps>
  <Step title="Install Docker">
    ```bash
    sudo apt install docker.io docker-compose-v2
    ```
  </Step>

  <Step title="Deploy OpenLDAP container">
    From the project root (where `docker-compose.yml` lives):

    ```bash
    # Start the container in detached mode
    docker compose up -d

    # Verify it's running
    docker ps
    ```
  </Step>

  <Step title="Add LDAP and VAPID keys to environment">
    Edit `backend/.env.production`:

    ```bash
    nano backend/.env.production
    ```

    Add the following variables:

    ```env
    # LDAP Configuration
    LDAP_URL=ldap://localhost:389
    LDAP_BASE_DN=dc=canaccesible,dc=es
    LDAP_ADMIN_DN=cn=admin,dc=canaccesible,dc=es
    LDAP_ADMIN_PASSWORD=kjhslIOP9002NSILXO

    # Web Push VAPID Keys
    VAPID_PUBLIC_KEY=BERgLLd2_B1K-eW4DIiGHufqPzReM8dRIZwRD1JmgCwdvImZpvK1hRF0x9Bi6_175zylm6AZ9wxYgGhGEHZpP9s
    VAPID_PRIVATE_KEY=qyBspyIIQshY-D2nx2SvB3TQHQcg4XIJt78Qmepq7js
    ```
  </Step>

  <Step title="Build the static frontend">
    ```bash
    cd frontend
    npm run build
    ```

    Move the generated `dist` folder to the Nginx web root:

    ```bash
    sudo mkdir -p /var/www/canaccesible
    sudo cp -r dist/* /var/www/canaccesible/
    ```
  </Step>

  <Step title="Apply the final Nginx configuration">
    Update `/etc/nginx/sites-available/canaccesible.es` to serve static files and proxy WebSockets:

    ```nginx
    server {
        listen 80;
        server_name canaccesible.es www.canaccesible.es;
        return 301 https://$host$request_uri;
    }

    server {
        listen 443 ssl; # managed by Certbot
        server_name canaccesible.es www.canaccesible.es;

        ssl_certificate /etc/letsencrypt/live/canaccesible.es/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/canaccesible.es/privkey.pem;
        include /etc/letsencrypt/options-ssl-nginx.conf;
        ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

        # Serve static frontend
        location / {
            root /var/www/canaccesible;
            index index.html;
            try_files $uri $uri/ /index.html;
        }

        # Proxy API requests
        location /api/ {
            proxy_pass http://127.0.0.1:85;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        # WebSocket support (Socket.io)
        location /socket.io/ {
            proxy_pass http://127.0.0.1:85;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
        }
    }
    ```

    Apply and verify:

    ```bash
    sudo nginx -t
    sudo systemctl restart nginx
    ```
  </Step>

  <Step title="Start backend with PM2 and populate data">
    ```bash
    # Start backend in production mode
    NODE_ENV=production pm2 start index.js --name "backend"

    # Run migrations and seeders (populates MySQL and LDAP)
    npm run db:migrate
    npm run db:seed
    ```
  </Step>
</Steps>

## Sprint 5: Updates & maintenance

<Steps>
  <Step title="Pull latest code and update dependencies">
    ```bash
    cd CanAccesible
    git pull origin main

    cd backend
    npm install

    cd ../frontend
    npm install

    cd ..
    ```
  </Step>

  <Step title="Update configuration">
    Edit `backend/.env.production` to reflect any new changes (for example, adding Resend API keys and removing obsolete SMTP configuration):

    ```bash
    nano backend/.env.production
    ```
  </Step>

  <Step title="Reset OpenLDAP">
    Remove the existing container (and volume if necessary) and recreate it:

    ```bash
    # Stop and remove the OpenLDAP container
    docker compose down

    # Start the container again in detached mode
    docker compose up -d
    ```
  </Step>

  <Step title="Re-initialize the database">
    ```bash
    cd backend

    # Apply the new schema
    NODE_ENV=production npx sequelize-cli db:migrate

    # Populate initial data
    NODE_ENV=production npx sequelize-cli db:seed:all
    ```
  </Step>

  <Step title="Redeploy the frontend">
    ```bash
    cd ../frontend
    npm run build

    # Update the served files
    sudo cp -r dist/* /var/www/canaccesible/
    ```
  </Step>

  <Step title="Restart the backend">
    ```bash
    pm2 restart backend
    ```
  </Step>
</Steps>

## Architecture summary

The final production architecture routes all traffic through Nginx:

| Incoming path | Handled by | Details |
| :--- | :--- | :--- |
| `/` | Nginx static files | React build from `/var/www/canaccesible` |
| `/api` | Backend via PM2 | Port 85 |
| `/socket.io` | Backend via PM2 | Port 85, with WebSocket upgrade headers |

The backend connects to:
- **Managed MySQL** — DigitalOcean
- **OpenLDAP** — Docker container on `localhost:389`
